# 实施计划

- [x] 1. 提取并固化安全的测试种子
  - 从未跟踪的备份数据库中只读提取 `mwb` 的4个大类、35个菜品、24个小料、关联顺序及1–40号牌。
  - 生成不含账号凭据、会话、队列和历史的可审查种子文件，并忽略本地备份数据库。
  - 校验种子数量、价格、排序和菜品—小料关系。
  - _需求：1、8_

- [x] 2. 重建关系型 SQLite 数据层
  - 建立账号设置、分类、菜品、小料、号牌、账单、菜品任务和快照关系表及索引。
  - 实现账号隔离查询、事务助手和每个号牌唯一未结算账单约束。
  - 移除运行时对整块 `workspace.data_json` 的业务依赖。
  - _需求：1、2、3、4、5、6、7、8_

- [x] 3. 实现空库自动初始化与显式重建命令
  - 空数据库首次启动时原子创建 `yue / 123` 和指定测试配置，账单、任务、历史、会话保持为空。
  - 非空数据库启动时只做非破坏性建表检查，不覆盖任何数据。
  - 增加带 `--confirm-reset` 保护的测试库重建命令，并确保它不进入容器启动流程。
  - _需求：8_

- [x] 4. 完成分类、菜单、小料和号牌管理 API
  - 实现分类新增、改名和排序；菜品改用稳定分类 ID。
  - 保留菜品、小料的新增、编辑、启停、删除和排序能力。
  - 实现号牌增删、使用中删除保护和短令牌生成。
  - _需求：1、2、8_

- [x] 5. 实现号牌账单与持续加菜 API
  - 为号牌第一次加菜自动建账单，再次加菜复用同一未结算账单。
  - 在事务中保存菜品、小料、价格、份数和分类快照并更新号牌总额。
  - 返回点菜页所需的空闲/进行中号牌状态及当前账单详情。
  - _需求：2、3_

- [x] 6. 实现后厨两步任务 API
  - 实现 `waiting → making → completed` 条件状态更新和精确时间记录。
  - 实现按菜品筛选、数量/份数统计及批量开始、批量完成。
  - 拒绝越级、重复和任何第三种后厨动作，并在提交后按账号实时广播。
  - _需求：4_

- [x] 7. 实现结算 API 与幂等保护
  - 提供未结算列表、账单详情和历史分页查询。
  - 未完成菜品时禁止结算；全部完成后固化金额、时间和状态。
  - 重复点击或多设备并发结算只生成一次结果，结算后释放号牌。
  - _需求：3、6、7_

- [x] 8. 实现短二维码、公开进度与收款码 API
  - 使用16位 Crockford Base32 令牌和 `/Q/` 短路径生成 M 级纠错 SVG 二维码。
  - 实现只读公开进度、号牌级 SSE、同菜排队位置和无历史泄露的空状态。
  - 实现收款码上传、替换、删除与公开读取，限制格式和2 MB大小。
  - 增加公开请求限速、无缓存和引用来源保护。
  - _需求：5_

- [x] 9. 改造经营统计与完整数据导出
  - 营业额、订单数、客单价和菜品销量全部改为按已结算账单统计。
  - CSV按菜品任务展开，JSON保留账单—菜品—小料层级。
  - 所有导出金额换算为元、保留两位小数且不附加币种文案。
  - _需求：7_

- [x] 10. 改造员工端状态管理与四项导航

  - 主导航调整为“点菜、出餐、结算、我的”，保持账号退出和PWA行为。
  - 保留现有绿色工业/实用主义设计令牌及响应式基础。
  - _需求：2、4、6_
