# 账号数据隔离设计

## 数据模型

新增 `workspaces` 表，以 `user_id` 为主键保存该账号完整业务状态：

```text
user_id, data_json, created_at, updated_at
```

`users` 和 `sessions` 继续使用原表。一个 SQLite 文件可以保存多个互相隔离的工作台。

## 迁移

数据库启动时检测 `workspaces` 是否已有数据。若为空但旧业务表存在数据，则选择 `yue`，否则选择最早注册账号，将旧设置、菜品、小料、队列和历史写入其工作台。其他已有账号在服务层补建空工作台。

## 服务边界

- 鉴权中间件从会话取得 `user_id`，并挂载对应工作台。
- 状态、订单、菜单、小料、设置和统计接口只操作挂载的工作台。
- SSE 客户端记录 `user_id`；变更只广播给相同 `user_id` 的连接。
- 注册创建空工作台；演示数据脚本只覆盖目标账号的工作台。

## 验证

- 构建和语法检查。
- 双账号 API 冒烟测试：菜单、设置、队列、历史互不可见。
- 旧数据库迁移验证：旧数据仅出现在 `yue` 工作台。
