# 账号数据隔离需求

## 范围

修复所有登录账号共享菜品、小料、设置、订单和经营历史的问题。保留同一账号多设备实时同步，并兼容已经上线的 SQLite 数据库。

## 验收标准

- 当用户注册时，系统应为该用户创建独立的空工作台和默认号牌设置。
- 当用户读取或修改状态时，系统应只访问当前会话所属用户的工作台。
- 当同一账号在多台设备登录时，系统应继续实时同步该账号的订单与菜单变化。
- 当不同账号同时在线时，系统不得向其他账号的 API 或 SSE 连接发送本账号的数据。
- 当旧版数据库首次启动新版程序时，系统应将原有业务数据归属给用户名为 `yue` 的账号；如果不存在 `yue`，才归属给最早注册账号。
- 当迁移完成后，系统应保留所有账号、密码哈希、原有菜单、订单、设置和历史记录。

## 非目标

- 不增加员工角色、门店成员邀请或跨账号共享。
- 不拆分成多个 SQLite 文件。
- 不改变登录、部署和备份入口。
